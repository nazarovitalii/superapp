# Деплой superapp (MrSQM web-клиент) в Coolify

Пошаговая инструкция. Домен клиента: **`sapp.mrsqm.com`**.

> ⚠️ **Главное правило (после инцидента 2026-06-10):**
> **Сборку НЕЛЬЗЯ запускать на нашем VPS.** Сборка Angular съедает всю память
> сервера и роняет общую Supabase (БД, админку, парсеры). 10 июня так и
> произошло — пришлось перезагружать весь сервер.
>
> Поэтому теперь образ собирает **GitHub** (бесплатно, на своих мощностях),
> а Coolify только **скачивает готовый образ и запускает** его. Запуск занимает
> ~30 МБ памяти — для VPS это безопасно.

---

## Как это устроено (общая картина)

1. Ты делаешь `git push` в `main` (или это делаю я).
2. **GitHub Actions** автоматически собирает Docker-образ и кладёт его в
   **GHCR** (GitHub Container Registry — бесплатное хранилище образов GitHub).
   Файл, который это делает: `.github/workflows/build-web-image.yml` (уже в репо).
3. **Coolify** настроен на тип **Docker Image** — он берёт готовый образ из GHCR
   и запускает nginx с уже собранным сайтом.

Адрес образа будет:
`ghcr.io/nazarovitalii/superapp-web:latest`

## ЧАСТЬ A. Один раз: проверить, что GitHub собирает образ

> Это уже настроено в коде. Нужно только убедиться, что сборка прошла, и сделать
> образ доступным для Coolify.

### A1. Дождаться первой сборки

1. Открой репозиторий на github.com → вкладка **Actions** (вверху).
2. Найди workflow **build-web-image**. После пуша он запустится сам.
   Если хочешь запустить вручную — открой его → кнопка **Run workflow** →
   ветка `main` → **Run**.
3. Дождись зелёной галочки (сборка идёт ~8–15 мин, это нормально).
   Если упадёт (красный крест) — открой лог, скопируй последние ~30 строк и
   пришли мне.

### A2. Сделать образ доступным (видимость пакета)

> По умолчанию GHCR-образ приватный — Coolify не сможет его скачать без логина.
> Проще всего на старте сделать его **публичным** (в образе нет секретов —
> только статика сайта и публичный anon-ключ Supabase, который и так виден
> в браузере).

1. На github.com → твой профиль (аватар справа вверху) → **Your profile** →
   вкладка **Packages**.
2. Открой пакет **`superapp-web`**.
3. Справа → **Package settings** → промотай вниз до **Danger Zone** →
   **Change visibility** → **Public** → подтверди.

> Если не хочешь делать публичным — скажи мне, опишу вариант с приватным
> доступом (нужен будет токен GitHub в Coolify). Для старта публичный — проще
> и безопасно.

## ЧАСТЬ B. Один раз: настроить Coolify

### B1. DNS — направить домен на сервер

1. Зайди в панель управления доменом `mrsqm.com` (там же, где заводил
   `app.mrsqm.com`).
2. Добавь запись:
   - **Тип:** `A`
   - **Имя/Host:** `sapp`
   - **Значение:** `51.83.197.222`
   - **TTL:** авто / 300
3. Сохрани.

### B2. Создать приложение из готового образа

1. Coolify → нужный **Project** → **+ New Resource** → выбери
   **Docker Image** (НЕ «Public Repository», НЕ «Private Repository» —
   именно **Docker Image**, чтобы Coolify ничего не собирал).
2. **Image:** впиши
   `ghcr.io/nazarovitalii/superapp-web:latest`
3. Если Coolify спросит логин в registry — для публичного образа **не нужен**,
   оставь пустым. (Сделал образ публичным в шаге A2.)
4. **Port (Ports Exposes):** `80`

### B3. Домен и HTTPS

1. В настройках приложения → поле **Domains** (или «FQDN») впиши:
   `https://sapp.mrsqm.com`
   (именно с `https://` — Coolify сам выпустит Let's Encrypt сертификат).
2. Включи **Generate SSL / Force HTTPS**, если есть переключатель.

### B4. Запустить

1. Нажми **Deploy**.
2. Теперь это **быстро** (~1–2 мин): Coolify просто скачивает готовый образ и
   запускает. **Память VPS почти не тратится — сервер не зависнет.**

### B5. Проверка

1. Открой `https://sapp.mrsqm.com` — должен загрузиться интерфейс (тема/вёрстка
   Super Productivity, лента MrSQM).
2. Замок (HTTPS) — зелёный/валидный.
3. Лента на `/mrsqm/feed` пока показывает мок-данные (auth не подключён — M-3).

## Обновления на каждый push (автодеплой)

Чтобы при каждом `git push` Coolify сам подтягивал свежий образ:

1. GitHub собирает новый образ `:latest` (workflow build-web-image).
2. В Coolify у приложения есть **«Deploy webhook»** (URL в настройках →
   Webhooks). Можно вызывать его в конце GitHub-сборки, чтобы Coolify
   передеплоил автоматически.

> Если хочешь полный автодеплой (push → собралось на GitHub → Coolify сам
> обновился) — скажи, я допишу в workflow вызов этого webhook. Понадобится
> один секрет (URL вебхука) — куда его положить, подскажу пошагово (в
> GitHub Secrets, не в код).

## Почему теперь не упадёт VPS

- Сборка (тяжёлая, прожорливая) — на серверах **GitHub**, не на нашем.
- На VPS приезжает **готовый** образ: внутри только nginx + статические файлы.
- Запуск nginx ест ~30 МБ. Общая Supabase и остальное в безопасности.
