## Решения

- **Дорожка реагирует на голос (Web Audio), не декоративная анимация** — пользователь
  явно хотел «видеть, что идёт по голосу раскадровка». Canvas (не DOM-бары) → ноль
  нагрузки на Angular CD при 60fps.
- **Добавлены ✕ (отмена) и таймер** сверх буквального запроса — без них «Слушаю…»
  выглядело бы оборванным; обе вещи есть в ChatGPT. Оставлены с пометкой «уберу, если не
  нужно».
- **Причину планшетного бага вычислил по коду**, не на устройстве — фикс по учебнику
  тач-скролла; финальная проверка — за пользователем на реальном планшете.
- Гоча зафиксирована в память: `checkFile` = prettier→eslint, `no-mixed-operators` хочет
  скобки, prettier их срезает → дробить арифметику на промежуточные переменные.

## Сессия 11:16 — фикс «галочка → пустое поле»

**Симптом:** записал голос, нажал ✓ — поле пустое, без ошибки.

**Диагностика:** эндпоинт `/chat/transcribe` жив (401 без токена). Причина — фронтовый
`transcribe()` **глотал любую ошибку** (`return ''`) → пользователь видел пустоту без причины.
Сильная гипотеза по корню: бэкенд всегда слал файл Whisper как `audio.webm`, а Safari/iPad
пишут `audio/mp4` → Whisper определяет формат по расширению → 400 → пусто.

**Фиксы (2 репо):**

- **superapp** `gpt-stream.service.transcribe()` — теперь БРОСАЕТ с деталью (текст ошибки
  бэкенда / HTTP-код); композер ловит → показывает причину, пустой результат → «Не расслышал».
  Сброс ошибки при новой записи. +2 unit-теста. Деплой `d9d374fe8`.
- **gpt** `transcribe.js` — имя файла по mimetype (`extFromMime`: mp4/mp3/ogg/wav/webm).
  Деплой `0b58719` (коммитил только свой файл — в gpt чужой WIP TL-9/TL-13).

**Важно:** gpt — общий main с параллельной сессией; `origin/main..HEAD` был пуст → мой пуш
чистый. TG gpt через свой `.env.local` дал 404 (нет/другой токен) → отправил через
superapp `.env.local` (общий бот/чат).

**Дальше:** пользователь перезаписывает голос; если снова пусто — на экране теперь будет
точная причина (например `Whisper 400: ...` или `HTTP 401`), по ней добью.
