## Решения

- **Кнопка share и ±ширина — через flex, а не ручной пересчёт.** Тулбар `nowrap`, селекты `flex:0 0 auto` с `min-width`, поиск `flex:1`. Любой fixed-элемент (share) появляется/исчезает → слабину автоматически отдаёт/забирает поиск. Не двигаем селекты вручную.
- **Высоту строки и ширину scope менял через локальный override токена/min-width**, не трогая глобальные значения (инбокс-задачи и другие селекты не задеты).
- **«Готово» закрывать через `MatMenuTrigger.closeMenu()`**, а не убирать `stopPropagation` с обёртки (он нужен, чтобы меню не закрывалось при выборе типа).
- **Избранное: клиент корректен (доказано тестом).** Если на проде иконка не заполняется — причина серверная (`save_property`/RLS), не клиент.

### Что сделали

**S-5: Голосовой ввод через Whisper (chat-page):**

- Кнопка 🎤 в композере чата: нажать → запись начинается, кнопка пульсирует красным; нажать снова → запись стоп → spinner → Whisper расшифровывает → текст вставляется в поле ввода.
- Бэкенд (`gpt`): новый файл `src/transcribe.js`, эндпоинт `POST /chat/transcribe` (authUser JWT), JSON-лимит поднят до 10mb.
- Фронтенд: `MediaRecorder` → `Blob` → `FileReader.readAsDataURL` → base64 → POST. В сервисе: `transcribe(blob)` + `_blobToBase64()`. В компоненте: `recording`, `transcribing` сигналы + `toggleMic()`. Cleanup в `ngOnDestroy`.

### Решения

- **Whisper вместо Web Speech API** — Web Speech средний на профессиональной лексике (DLD, off-plan, JVC); Whisper-1 с `language=ru` точнее, цена ~$0.006/мин.
- **base64 через JSON, не multer** — избежали зависимости `multer` на бэкенде; для коротких голосовых (~5–30 сек) размер в норме.
- **mimeType с fallback** — `['audio/webm;codecs=opus', 'audio/webm', 'audio/ogg', 'audio/mp4'].find(isTypeSupported)` — поддержка Chrome/Electron/Safari.
