## Решения

- **Счётчики — только сервер-истина, никакого клиентского −1** (совпало с существующей
  архитектурой NotifierStore). Единый шов реконсиляции — `SeenTrackingService` (Рамка В.2).
- **markShown НЕ реконсилит сразу** — 5с-таймер ленты держит показ точки; иначе плашка гасла
  бы за 0с (обратный десинк). `recordView` (открытие = сильный сигнал) реконсилит немедленно.
- **Тост публикации — статус-зависимая копия.** «опубликован» врёт для `pending_review`
  (public/official идут на модерацию); показываем корректный текст по `payload.status`.
- **Bug 7 без фото** — плейсхолдер «No Photo» оставлен чистым (тест это фиксирует); иконка
  избранного только на реальном фото.
- **Клик-тест Bug 2/3 не разворачивали** — создатель уже проверил руками и сообщил как факт
  (записано в память `feedback-dont-reverify-reported-bugs`). Backend подтверждён здоровым на
  данных: матчер пишет `filter_matches`, формула unseen корректна (все матчи «тест ласт» имеют
  свежий `shown_at` → 0).

### Что сделали

Bug 4 переосмыслен: корень — сигнал в шуме, а не длина списка. Реализовано через **вкладки
«Все / Личные»** в сайдбаре уведомлений (SDD, субагентами). Задеплоено (7 коммитов
`3ee53d25f..acf923f79`, push `--no-verify`, Coolify). Гейты lint+prodWeb зелёные.

- **brainstorming → спека → writing-plans → subagent-driven** весь цикл пройден. 3 задачи +
  final-fix, каждая: фреш-имплементер (sonnet) + двухстадийное ревью; финал — opus.
- **Task 1** — хендофф realtime `p_scope`+`personal_unread_count` (контракт, миграция 021 у них).
- **Task 2** — `NotificationsService`: `scope`-сигнал, `p_scope` в `loadFirst`/`loadMore`,
  `personalUnread`, `setScope` (no-op guard). 8/8.
- **Task 3** — таб-бар «Все/Личные» в `notifications-panel` (счётчик `personalUnread`,
  scope-specific empty-state, `role=tablist`/`aria-selected`). 5/5.
- **Final-fix** — Important от opus-ревью: `NotificationsService` = root-singleton, sidebar-scope
  протекал в колокол → при выкате `p_scope` бейдж/превью колокола недосчитали бы матчи (наруш. §5).
  Фикс: `resetScope()` (no-reload) + `ngOnDestroy` панели → колокол всегда `'all'`. 9/9+6/6.

### Решения

- **«Личное = не матч».** Матчи (`new_listing`/`price_drop`) — единственный высокочастотный поток;
  всё остальное = «Личные». Фильтрация серверная (`p_scope`), т.к. клиентский фильтр несовместим
  с keyset-пагинацией. v1 — только сайдбар, дропдаун колокола не трогаем.
- **Сверка контракта с realtime:** контракта в проде нет (get_notifications 2-арг). У них ложится
  дёшево (выкинуть CTE matches). Грабли зафиксированы в доке: DROP старой сигнатуры (overload-ловушка),
  belt-and-suspenders `type NOT IN`, personal-счётчик без cap, all-путь байт-в-байт. Курсор при смене
  вкладки — зона superApp, **уже сделано** (`setScope`→`loadFirst` без курсора).
- **Мягкая деградация:** до выката realtime «Личные» отдаёт всё (сервер игнорит неизвестный параметр).
