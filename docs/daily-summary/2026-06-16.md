## Сессия 3 — лайтбокс галереи + миграция «своих объектов»

### Что сделали

- **Полноэкранный лайтбокс галереи** (`property-gallery-lightbox`) — клик по фото в
  карточке открывает большой просмотр в стиле focus-mode оверлея: крестик справа,
  стрелки ←/→, счётчик, полоса миниатюр, клавиатура (←/→/Esc), свайп на тач. Свой
  компонент, без внешних библиотек, токены SP. 6 unit-тестов. Задеплоено
  (origin/main `61bbe4f83`).
- **Написана миграция** `docs/migrations/2026-06-16-property-owner-actions.sql` — 3
  SECURITY DEFINER RPC для действий владельца над своим объектом: `update_property`
  (только цена+описание), `actualize_property` (last_actualized_at=now), `archive_property`
  (status archived_sold/withdrawn). Каждая проверяет owner_id=auth.uid(). **Ещё НЕ
  применена** — ждёт применения в Studio.

### Решения / выясненное

- На `properties` нет UPDATE-RLS (только insert/select) → агент не может менять объекты
  с клиента. Поэтому действия владельца = узкие RPC (безопаснее общей UPDATE-политики).
- История цены (`previous_price`) — оставлена существующим триггерам БД, RPC её не трогает
  (тело `log_property_changes` в доках не раскрыто, не рискуем конфликтом).
- Поле «планировка» (layout) в карточке — **на паузе** по решению создателя.
- Превью фото в строке ленты — отдельной задачи нет; при желании попробуем позже.
