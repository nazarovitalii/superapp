## Решения

- **Бейдж агента не показываем** — система бейджей вне MVP (правила `mrsqm.md`),
  хотя `get_property` отдаёт `agent.badge_level`.
- **Контакты агента не гейтим на клиенте** — БД сама отдаёт `whatsapp_phone`/
  `broker_license` по правам (свой/Pro/сеть), иначе NULL. Клиент логику не дублирует.
- **Статистика в карточке** — дата + просмотры (по выбору создателя).
- **Превью фото в строке ленты** — отложено (отдельный шаг), сейчас только карточка.
- **`layout_id`** — резолв требует знания комьюнити (`community_layouts`), в карточке
  пропускаем; остальные id-массивы резолвятся глобальным `get_filter_options`.
