## Сессия 3 — галерея переведена на Swiper.js (MIT)

### Что сделали

- **ng-gallery заменён на Swiper.js (MIT v12.2.0).** Создатель выбрал Swiper как наиболее
  готовое MIT-решение без платного лицензирования (lightGallery = GPL/коммерческая).
- **Карточка:** стрелки prev/next поверх фото (position: absolute, opacity 0 → 1 при `:hover`
  на `.gallery`, плюс hover-затемнение самой кнопки). Счётчик «N / M» снизу-справа.
  Вся навигация сохранена через сигналы `prevPhoto()`/`nextPhoto()` без Swiper JS в карточке.
- **Лайтбокс:** fullscreen overlay (`@if lightboxOpen()`), два Swiper-инстанса — основной
  (большое фото) + thumbs (полоса 60×45 миниатюр снизу). Swiper инициализируется через
  `afterNextRender({ injector })` после рендера `@if`. Escape-handler через `@HostListener`.
- ng-gallery удалён из `package.json`; Swiper CSS добавлен в `angular.json` styles.
- Билд: зелёный. 10/10 тестов. Задеплоено.

### Решения

- **Swiper, не PhotoSwipe** — Swiper имеет встроенный Thumbs-модуль, требует минимум
  кастомизации. PhotoSwipe удобнее для фото-сайтов, но thumbs-strip пришлось бы делать вручную.
- **Vanilla JS Swiper** (не `swiper/angular`, не Web Components) — `swiper/angular` deprecated
  в v9+, Web Components требуют `CUSTOM_ELEMENTS_SCHEMA`. Vanilla JS + `@ViewChild` чище.
- **Lightbox через `@if`**, не через hidden div — Swiper не инициализируется на `display: none`,
  `@if` + `afterNextRender` решает проблему без утечек памяти.
- Lightbox рендерится внутри `.detail-panel` с `position: fixed` — CDK Overlay не нужен,
  `z-index: 9999` достаточно (панель не имеет `transform` в этот момент).

### Где остановились

- Galерея на Swiper.js задеплоена. Ждём проверки в браузере создателем.

## Сессия 4 — диагностика «галерея не обновляется» + реальный фикс

### Что сделали

- **Долгая ложная диагностика про кэш.** Создатель видел «старую/сломанную галерею»
  даже после редеплоя и очистки данных сайта. Я несколько ходов грешил на кэш браузера
  и service worker. Детерминированная проверка прода (`curl` styles/headers): код **новый**
  (Swiper CSS на месте), CDN нет (nginx), деплой прошёл. То есть кэш был не при чём.
- **Нашли настоящий баг.** «Слой ленты поверх галереи» = лайтбокс `position:fixed` заперт
  внутри `right-panel-content`, у которого `will-change: transform` (создаёт containing-block
  для fixed). Overlay не покрывал экран, лента наезжала. Тот же баг, что был у самого первого
  самодельного лайтбокса.
- **Фикс лайтбокса:** нативный `<dialog>` + `showModal()` → рендер в **top layer** браузера,
  выше любого z-index и transform-контекста. Escape/backdrop нативные, `@HostListener` убран.
  10/10 тестов, билд зелёный.
- **Убит service worker в прод-вебе** (`productionWeb serviceWorker: false`): он намертво
  кэшировал ассеты/фото (externalAssets-группа отдавала битые фото из кэша — отсюда «пропали
  все фото в листингах»). + safety-worker по `/ngsw-worker.js` авто-чистит застрявшие браузеры.

### Решения

- **`<dialog>` top layer вместо CDK Overlay** — тот же эффект «вынести из transform-контекста»,
  что давал ng-gallery через `<body>`, но без зависимостей. Safari 15.4+.
- **PWA-офлайн не нужен** MrSQM (онлайн-only, данные из Supabase). Установку на рабочий стол
  не теряем — она на manifest + apple-touch-icon, SW не требуется. Веб-пуш (iOS 16.4+) — если
  понадобится, вернём минимальный SW, не кэширующий монстр.
- **Урок по диагностике:** не залипать на гипотезе «кэш» — сначала детерминированно сверить,
  что реально на проде (curl бандлов/заголовков), и просить у создателя точное описание симптома
  («лента поверх галереи» сразу указало на transform-trap).

## Сессия 5 — настоящий баг галереи найден через Playwright на проде

### Что сделали

- **3 раунда деплоя «на ощупь» по галерее** (лента поверх → одни фото → чёрное окно), все мимо.
  Симптом «миниатюры есть, фото нет, правой стрелки нет» — гадал про кэш и нулевую высоту, оба
  неверно.
- **Зашёл на прод через Playwright** (логин создателя), померил DOM и нашёл РЕАЛЬНУЮ причину:
  `.lightbox-main` — флекс-элемент внутри `<dialog>`, его ширина бралась от контента Swiper, а
  Swiper брал ширину от элемента → петля, раздувавшая слайд до **16 777 216px (2²⁴)**. Фото
  грузилось и рисовалось, но улетало на x≈8.4М px за экран; миниатюры (фикс-ширина) оставались
  видны — отсюда асимметрия симптомов.
- **Фикс:** `width: 100vw` + `max-width: 100vw` + `min-width: 0` на `.lightbox-main` (100vw не
  зависит от сломанного containing-block, разрывает петлю). **Проверил фикс вживую на проде**
  (inject через Playwright) ДО деплоя: ширина 1352, фото центрировано, стрелки на экране.
- Задеплоил, дождался выкатки образа, **перепроверил на свежей загрузке** (новый бандл,
  без патчей): фото на весь экран, счётчик, стрелки ‹ ›, 3 миниатюры, листание 1/3→2/3 — скриншот.
- Фикс «одни фото» тоже подтверждён: объекты без фото честно показывают заглушку.

### Решения / уроки

- **Главный урок:** при визуальных багах — лезть в живой DOM с самого начала (Playwright + замер
