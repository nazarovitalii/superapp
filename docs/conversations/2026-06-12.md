## Идеи: как внедрить планировщик в вашу CRM

Самое ценное в этом форке — вы получили готовый планировщик задач «бесплатно». Для риелтора это превращает MrSQM из «ленты объектов» в рабочий стол на весь день. Идеи от простого к сложному:

1. **«Задача из объекта»** (самое простое и сильное). В hover-кнопки и в карточку объекта добавить «➕ В задачи»: создаётся задача Super Productivity с названием «Позвонить по: Marina Heights, 2BR, 1.4M» и ссылкой на объект. Риелтор утром собирает из ленты план на день, потом работает по списку Today с таймером.
2. **Проект = клиент.** В Super Productivity есть проекты — завести «Проект» на каждого покупателя: внутри задачи «подобрать варианты», «показ в субботу», подзадачи-объекты. Это, по сути, мини-CRM по сделкам без единой новой таблицы.
3. **Показы в календаре.** У задач есть план на время (`dueWithTime`) и напоминания — показ объекта назначается как задача со временем, и встроенный планировщик дня сам строит расписание показов с конфликтами времени (эта логика уже есть).
4. **Повторяющиеся задачи для рутины риелтора:** «обновить свои листинги» раз в неделю, «прозвонить тёплых клиентов» по понедельникам — механизм repeat уже встроен.
5. **Выбранные чекбоксами объекты → пачка задач.** Раз мы сделали множественный выбор: кнопка «Создать задачи по выбранным» — каждая запись становится задачей в проекте клиента. Это естественное «массовое действие» для плашки «Выбрано: N».
6. **Учёт времени для тимлидов агентства** (дальний прицел): таймеры и worklog Super Productivity дают отчёт «сколько времени агент тратит на сделку» — полезно для B2B-подписки агентств.

# /daily-summary — резюме рабочего дня

## Что делает

Создаёт `docs/daily-summary/YYYY-MM-DD.md` с кратким резюме сессии. Максимум 300 строк.
Фокус на решениях и контексте, не на пересказе кода.

## Структура

```markdown
# YYYY-MM-DD

## Что сделали

- ...

## Решения

- ...

## Где остановились

...

## Что дальше

- ...
```

## Алгоритм

1. Дата: `date +%Y-%m-%d`
2. Анализ переписки + git log за день:
   ```bash
   git log --oneline --since="today 00:00" --until="tomorrow 00:00"
   ```
3. Резюме — результаты, ключевые решения и почему, где остановились, следующие шаги.
4. Сохранить:
   - Нет файла → создать с заголовком `# YYYY-MM-DD`.
   - Есть → дописать в конец (`---` + `## Сессия HH:MM`), не перезаписывать.
5. Коммит — через deploy-ритуал (шаг 6).

## Gotchas

- **Лимит 300 строк** — не влезает → резать детали.
- **Есть файл — дописывать**, не перезаписывать.
- **Не дублировать** `docs/conversations/` — там сырьё, здесь суть.
