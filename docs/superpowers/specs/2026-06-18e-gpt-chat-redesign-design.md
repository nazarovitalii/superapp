# Спека — редизайн таба AI Chat (S-3, «ChatGPT-вид»)

**Дата:** 2026-06-18
**Статус:** дизайн согласован, готов к плану реализации
**Связано:** S-2 (базовый таб, `2026-06-18c-gpt-chat-tab-design.md`) — это его визуальный редизайн.
Бэкенд-контракт и сервис `gpt-stream.service` НЕ меняются.

---

## 1. Цель

Переделать таб AI Chat (`/mrsqm/chat`) так, чтобы он выглядел и ощущался «как ChatGPT»,
**оставаясь в визуальном языке Super Productivity** (токены/темы SP). Сейчас — рабочий, но
базовый пузырьковый чат. Нужно: центр-колонка с воздухом, ассистент текстом во всю ширину
(без пузыря), аккуратный композер-пилюля, плавный стрим, чипы-подсказки на пустом экране.

## 2. Согласованные решения (из брейншторма)

- **Направление:** «ChatGPT-UX в теме SP» — раскладка/UX как у ChatGPT, но отрисовано
  токенами SP. НЕ пиксель-копия ChatGPT (это был бы чужой визуальный язык — против правила
  проекта). НЕ просто полировка текущих пузырей.
- **Доп. фичи:** только **чипы-подсказки** на пустом экране. Копирование/регенерация/
  лайки/редактирование — осознанно НЕ делаем (YAGNI; тривиально добавить позже).
- **Структура:** переписать **на месте** один компонент `chat-page` (template + SCSS +
  минимум TS). Без дробления на под-компоненты (лишние абстракции для одной страницы) и без
  сторонних чат-либ (Shadow-DOM, чужой визуал, токены SP не проходят).

## 3. Архитектура

Остаётся один standalone OnPush-компонент `ChatPageComponent`
(`src/app/mrsqm/pages/chat/chat-page.component.{ts,html,scss}`) + его спека. Сервис
`GptStreamService` (`streamMessage` / `loadHistory` / `sendNonStreaming`) и контракт
(`{ text, channel:'web' }` + `Authorization: Bearer`, история через `GET /chat/history`)
**не трогаем**. Меняется только презентационный слой + немного логики UI.

### Состояние (signals)

Сохраняются: `messages`, `status`, `streaming`, `loadingHistory`, `error`, `draft`.

Добавляются:
- `pinnedToBottom = signal(true)` — лента «приклеена» к низу. При стриме, если `true`, авто-скролл
  вниз; кнопка «↓ вниз» видна только когда `false`.
- Константа (не signal) `SUGGESTIONS: { label: string; prompt: string }[]` — 4 риелторских
  примера (`label` = короткая подпись на чипе, `prompt` = что уходит в `send`):
  1. «2BR в Dubai Marina до 2M» → «Покажи 2BR квартиры в Dubai Marina до 2 млн AED»
  2. «Аренда виллы в Arabian Ranches» → «Что есть в аренду — виллы в Arabian Ranches»
  3. «Off-plan с рассрочкой» → «Подбери off-plan проекты с рассрочкой от застройщика»
  4. «1BR: JVC vs JLT» → «Сравни цены на 1BR квартиры в JVC и JLT»

  Тексты не финальны на уровне продукта, но в спеке зафиксированы — правятся одной строкой.

### Новые методы

- `sendSuggestion(prompt: string)` — `draft.set('')` + `send(prompt)`.
- `onMessagesScroll(event)` — вычисляет «близко ли к низу» (порог ~80px), пишет `pinnedToBottom`.
- `scrollToBottom()` — программно прокручивает контейнер ленты вниз (через `viewChild` на
  элемент ленты), ставит `pinnedToBottom(true)`.
- авто-скролл при новых токенах: после `onToken`/`onDone`, если `pinnedToBottom()` — прокрутка
  вниз. Реализация — `afterNextRender`/`effect` на `messages` + ref ленты (без прямого DOM-доступа
  в обход Angular; ref через `viewChild`).
- авто-рост поля ввода: в `onInput` — сброс `height='auto'` → `height=scrollHeight` (cap по
  max-height из CSS). Поле — `viewChild` на textarea.

Все ссылки на DOM — через `viewChild()` (правило проекта: никакого прямого DOM-доступа).
Подписки/стрим — как в S-2 (AbortController, очистка в `ngOnDestroy`).

## 4. Визуальный дизайн (по секциям)

### A. Раскладка
- Центр-колонка `max-width: 768px`, по центру, во всю высоту таба.
- `chat-messages` — скролл-контейнер (`flex:1; overflow-y:auto`), `(scroll)="onMessagesScroll"`.
- Композер прижат снизу (`flex-shrink:0`), отделён тонкой границей `--extra-border-color`.
- Кнопка **«↓ вниз»** — круглая, фикс. над композером по центру колонки, видна при `!pinnedToBottom()`.

### B. Пустое состояние (`messages().length === 0 && !loadingHistory()`)
- По центру колонки: крупный бот-глиф (`mat-icon smart_toy`, primary-tint, ~56px), заголовок
  «Чем помочь по рынку Дубая?», короткий подзаголовок.
- Ниже — сетка из **4 чипов** (`SUGGESTIONS`). Чип = кликабельная «карточка-пилюля» в стиле
  SP (фон `--task-c-bg`, бордер `--extra-border-color`, радиус `--task-border-radius`, hover —
  лёгкий primary-акцент). Клик → `sendSuggestion(prompt)`.

### C. Сообщения (`@for ... track $index`)
- **Ассистент:** контейнер во всю ширину колонки, слева — маленький бот-маркер (кружок с
  `smart_toy`, primary-tint). Текст — `<markdown [data]="m.text">`, БЕЗ фонового пузыря.
  Markdown-типографика в теме SP (наследуем правила из текущего SCSS: списки, `code`, `pre`,
  + аккуратные таблицы). Цвет `--text-color`.
- **Юзер:** мягкий пузырь справа, фон `--c-primary`, текст `--palette-primary-contrast-500`,
  `max-width: ~75%`, радиусы как сейчас. `white-space: pre-wrap`.
- Появление каждого сообщения — `@keyframes` fade + slight slide-up (≤150ms, `prefers-reduced-motion`
  отключает).

### D. Стрим и статус
- Индикатор печати — вместо мигающего `|` ChatGPT-style: пульсирующая точка/каретка в конце
  ответа (`m.streaming === true`). Анимация мягкая.
- Tool-статус (`status()`, из `TOOL_LABELS`, напр. «Ищу объявления…») — тонкая строка с
  **shimmer**-градиентом (бегущий блик по тексту), показывается над/у печатающегося ответа.
- Кнопка **стоп** в композере во время `streaming()` (как сейчас), курсор/индикатор гаснет на
  `stop()`/`onDone`/`onError` (логика S-2 сохранена).

### E. Композер
- Обёртка-«пилюля» во всю ширину колонки: фон `--task-c-bg`, бордер `--extra-border-color`,
  большой радиус, `:focus-within` — primary-бордер.
- Внутри: авто-растущая `textarea` (min 1 строка ~44px, max ~6 строк / 160px, дальше внутренний
  скролл), плейсхолдер «Напишите сообщение…».
- Кнопка отправки — круглая, **внутри** пилюли справа (`send`-иконка), disabled при пустом
  `draft().trim()`; при `streaming()` заменяется на круглую **стоп**-кнопку.
- Enter — отправить; Shift+Enter — перенос (логика `onKeyDown` из S-2).

### F. Тема / анимации
- Только токены SP — тёмная/светлая работают, таб не выбивается из приложения. Никаких
  хардкод-серых «под ChatGPT».
- Все анимации уважают `@media (prefers-reduced-motion: reduce)`.
- Никаких `.mat-*`/`.mdc-*`-оверрайдов и рестайла shared-компонентов (правило стайл-ревью).
  `::ng-deep markdown` внутри ленты — допустимо (прецедент в feed/property-card).

## 5. Тесты (Jasmine/Karma, co-located spec)

Сохранить зелёными существующие 8 тестов S-2. Добавить:
- клик по чипу-подсказке вызывает `streamMessage` с текстом подсказки и очищает пустое состояние;
- `onMessagesScroll` у низа → `pinnedToBottom()===true`; отлистывание вверх → `false`;
- авто-рост: `onInput` меняет высоту textarea (или хотя бы вызывает ресайз-логику без ошибок);
- пустое состояние показывает 4 чипа; при `messages().length>0` чипы скрыты.

DOM-тесты — через `fixture.nativeElement.querySelector`, как в текущей спеке.

## 6. Критерии успеха

- Таб визуально читается как «ChatGPT» (центр-колонка, ассистент без пузыря, пилюля-композер,
  чипы), но в тёмной И светлой теме SP выглядит родным.
- Стрим печатает плавно, авто-скролл следует за низом, кнопка «↓ вниз» появляется при отлистывании.
- История подтягивается при открытии (S-2 поведение не сломано).
- `npm run checkFile` чисто по .ts/.scss; юнит-сюит зелёный; прод-сборка без ошибок.

## 7. Вне объёма (YAGNI)

Копирование ответа, регенерация, лайк/дизлайк (нужен эндпоинт), редактирование сообщений,
картинки-аватары юзера, вложения/файлы, голосовой ввод, смена модели, сайдбар-история тредов,
любые правки бэкенда/БД. Пиксель-копия ChatGPT (чужой визуальный язык).

## 8. Риски

- Авто-скролл + авто-рост легко сделать «дёргающимися» — порог близости к низу и `afterNextRender`
  убирают рывки; проверить на длинной ленте и быстром стриме.
- `track $index` при стриме (последний пузырь мутирует) — оставляем как в S-2 (работает, т.к.
  обновляется именно последний элемент новым объектом).
- Хот-пас тут не действует (это не task.component), но лента может быть длинной — следить за
  отсутствием тяжёлых выражений/геттеров в шаблоне.
