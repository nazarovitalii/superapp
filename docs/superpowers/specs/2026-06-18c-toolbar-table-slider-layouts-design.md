## Запросы (вербатим → дизайн)

### Бегунок адреса · `pages/add-property/add-property-page.{html,ts,scss}`

- **U-0a.** Тащить мышкой/пальцем, не только клик по точкам. Видимый «thumb»-кружок на активной точке;
  `pointerdown` на треке/thumb → захват, `pointermove` пересчитывает индекс из X относительно ширины
  трека, `pointerup` снапит к ближайшему уровню. Клик по точкам (`selectReveal`) сохраняется.
  Нижняя граница — `communityIndex()` (ниже комьюнити нельзя). Pointer events = мышь + тач одним кодом.

### Тулбар · `pages/feed/feed-page.{html,scss}`

- **U-1.** Все контролы одной высоты = строка объекта **40px** (`--task-first-line-min-height`; сейчас
  селекты/иконки 36px). Текст 14px, `font-weight 400`, белый фон `--task-c-bg`, скругление
  `--task-border-radius` (часть уже есть — добиваем высоту и иконко-кнопки).
- **U-8.** Селект охвата «All Inventory» — фиксированная ширина под самый длинный пункт «Friends
  Inventory» (не меняет размер при переключении). `min-width` на `.scope-select`.
- **U-10.** Убрать `.toolbar-spacer`; `.tb-search` растягивается (`flex-grow`) и съедает гап — сегмент/
  сделка/тип/сорт/фильтр пакуются вправо без дыры между типом и сортировкой.
- **U-11.** Иконки сортировки и фильтра — в квадратной рамке как селекты: бордюр `--extra-border-color`,
  фон `--task-c-bg`, скругление `--task-border-radius`, 40×40. Новый класс `.tb-icon-btn` (заменяет
  `mat-icon-button`-стиль для этих двух кнопок).

### Тип-дропдаун · `pages/feed/feed-page.{html,scss}` (+ `feed-page.component.ts` local signal)

- **U-2.** Одна колонка ~240px. Локальный сигнал `typePanelCat` (`'residential' | 'commercial'`,
  дефолт = `filter.category() ?? 'residential'`) переключает таб. Структура:
  - Ряд табов Residential / Commercial (выбор только меняет, что показано; не применяет фильтр сам по себе).
  - Строка «Все {категория}» (`selectCategoryAll`).
  - Список `unit`-типов активной категории; активный тип раскрывает `.type-subs` чипами под собой.
  - Низ: Сбросить (`clearType`) / Готово (закрывает меню).
    Единый кегль 14px для типов/«Все»/чипов; заголовки-табы — semibold. Убрать старую 2-колоночную сетку
    (`.type-cols`/`.type-col`) и `width:560px` → `width:~240px`.

### Строка-карточка · `components/property-card/property-card.{html,scss}` + шапка `feed-page.scss`

- **U-7.** Убрать `type-handover` из `col-type`. В `col-address`: строка1 = `addr-leaf` + (если
  `handover==='offplan'`) инлайн-метка `(off-plan)` серым/мельче; строка2 = community серым (как сейчас).
- **U-6.** `col-type` (теперь только имя типа) — разрешить перенос на 2 строки: снять `white-space:nowrap`
  у этой колонки, `line-height` плотный, центрирование сохранить.
- **U-9.** Вернуть вторую строку в `col-size`: line1 = `area_sqft` (BUA, как есть), line2 = `plot_sqft`
  серым (если не null), число без «sqft» (консистентно с BUA). `col-size` → flex-column как `col-beds`.
- **U-4.** Формат даты `last_actualized_at || published_at`: `Today` / `Yesterday` / `16 June`; если год
  даты < текущего → `16 June 2025`. Реализация — `computed`-сигнал `dateLabel` в `property-card.ts`
  (карточка = hot path; **никаких функций/геттеров в шаблоне**, см. CLAUDE.md про task.component).
  Логику формата вынести в чистую функцию `util` + юнит-тест (Today/Yesterday/этот год/прошлый год).
- **U-5.** Выровнять заголовки шапки (`.th-*`) под выравнивание ячеек карточки: тип/beds/площадь/цена/
  агент — по центру; адрес — слева; дата — справа. Сейчас `th` правит beds/size/price/agency/date вправо,
  а ячейки центрированы → разъезд. Сетка `grid-template-columns` шапки и `.first-line` уже идентичны —
  меняется только `text-align` заголовков.

## Критерии успеха

Каждая правка проверяема: бегунок тащится мышью и снапит к уровню (не ниже комьюнити); тулбар-контролы
одной высоты 40px, белые, скруглены; «All Inventory» не прыгает; гапа между типом и сортировкой нет;
иконки сорт/фильтр в рамке; тип-дропдаун узкий (одна колонка), тексты выровнены; в строке «(off-plan)»
у off-plan-объектов в адресе, у ready — пусто; длинный тип переносится на 2 строки; в «Площади» вторая
строка plot; дата `Today`/`Yesterday`/`16 June`/`16 June 2025`; заголовки шапки над своими колонками;
сортировка работает (или выписан серверный гейт). `checkFile` зелёный; юнит-тесты на дату/бегунок/
сортировку; полный сюит зелёный.
